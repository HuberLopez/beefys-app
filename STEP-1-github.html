Roadmap › Phase 1 — Put it online

Step 1:
Get it on the internet

Follow this top to bottom and your app gets a real link anyone can open on their phone. No coding, no command line, no credit card. Keep this open on one side while you click along on the other.

B
~30 minutes $0 No coding Just clicking
The plan: put your files on GitHub (free online storage), then connect Netlify (free hosting) to it. GitHub holds the files; Netlify turns them into a live website. Do them in that order. There's an even-simpler GitHub-only option at the bottom if you'd rather skip Netlify.
A

Put your files on GitHub

~10 minutes

  1. Make a free GitHub account

    Go to github.com → Sign up. Use your email, pick a username (e.g. comebeefys), confirm the email. Free, no card.

  2. Create an empty repository

    Top-right, click the + → New repository.

    Repository name: beefys-app. Leave it Public. Don't tick any of the "add a README / .gitignore" boxes. Click Create repository.

  3. Upload your files

    On the new empty repo page, click the link "uploading an existing file" (or Add file ▾ → Upload files).

    Open your beefys-app folder on your computer, select everything inside it (index.html, the other .html files, the assets and tokens folders, styles.css, README.md) and drag it all onto the page.

    ⚠︎
    Drag the files that are inside the folder — not the folder itself. index.html must end up at the top level of the repo, or the site won't open.

  4. Save it

    Scroll down and click the green Commit changes button. Your files are now online. 🎉 (You'll see them listed on the repo page.)

B

Turn it into a live website with Netlify

~10 minutes

  1. Sign up for Netlify with GitHub

    Go to netlify.com → Sign up → choose "Sign up with GitHub". Click Authorize when it asks. This links the two accounts so Netlify can see your files.

  2. Import your repository

    Click Add new site → Import an existing project → Deploy with GitHub. Pick beefys-app from the list. (If you don't see it, click "Configure Netlify on GitHub" and give it access.)

  3. Don't change any settings — just deploy

    It'll ask for a "build command" and "publish directory". Leave them blank (publish directory can stay as . or empty). This is a plain static site — there's nothing to build. Click Deploy.

  4. Open your live site

    Wait ~30 seconds. Netlify gives you a link like random-name-123.netlify.app. Click it — that's your app, live on the internet.

    To rename it: Site configuration → Change site name → e.g. comebeefys → you get comebeefys.netlify.app.

    ★
    Bonus: every time you upload new files to GitHub, Netlify re-publishes automatically. Update once, live everywhere.

Prefer to skip Netlify? (GitHub Pages — fully free, no second account)

GitHub can host the site by itself. Slightly less convenient for a custom domain, but zero extra signups:

  1. In your beefys-app repo, click Settings → Pages (left sidebar).
  2. Under "Source" pick Deploy from a branch, branch main, folder / (root) → Save.
  3. Wait 1–2 minutes, refresh — a green box shows your URL like comebeefys.github.io/beefys-app.
C

Optional: your own domain

later · ~$12/year

  1. Buy the domain

    Want pedidos.comebeefys.com instead of a netlify.app link? Buy it at Namecheap (~$10–15/year). Skip this entirely until you're ready — the free link works perfectly for launching.

  2. Point it at your site

    In Netlify: Domain settings → Add a domain → type your domain → follow the one-page instructions it gives you. It walks you through it.

That's it — you're online

Your app now has a real link you can text to anyone and open on a phone. Click through Cliente → Operador → Backend on your actual phone to confirm it all works.
